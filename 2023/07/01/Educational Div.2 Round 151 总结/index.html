C 「Strong Password」

考虑贪心策略，每一次从限定条件中选取最晚在 database string 中出现的字符，这样可以尽可能快地消耗掉 database string 的前缀，如果中途出现了后缀中不存在的限定条件内的字符，则说明存在答案，反之说明不存在。

为了记录 database string 所有后缀中字符的出现时间，我们需要一个数组来维护每个后缀所有字符出现的最小索引。

时间复杂度：\(O(nm)\)

空间复杂度：\(O(nm)\)
